# ldsc_cts_discrete

A Snakemake workflow for **cell-type-specific LDSC (CTS)** using **discrete BED peak annotations**.

This repository is intentionally minimal and reusable:
- it **does not** perform munging;
- it expects **LDSC-ready `.sumstats.gz`** files;
- it auto-discovers traits if `traits: []` in the config;
- it automatically generates the `.ldcts` file;
- it validates the main input paths at startup.

## Requirements

You need:

- Python ≥ 3.10
- Snakemake
- bedtools
- pybedtools
- pandas
- numpy
- LDSC

The easiest way to install these dependencies is using **mamba / conda**.

### 1. Create the conda environment

Move to the repository:

cd ldsc_cts_discrete

Create the environment:

mamba env create -f environment.yml

Activate the environment:

mamba activate ldsc_cts

### 2. Install LDSC

Clone the LDSC repository:

git clone https://github.com/bulik/ldsc.git

This creates a folder:

ldsc/

In config/config.yaml, set the path to ldsc.py:

ldsc_py: "/path/to/ldsc/ldsc.py"

Example:

ldsc_py: "/home/user/tools/ldsc/ldsc.py"




## What the workflow does

1. Split each BED file by chromosome
2. Build LDSC `.annot.gz` files from chromosome-specific BEDs
3. Compute LD scores for each cell type
4. Generate a single `all.ldcts` file automatically
5. Run `ldsc.py --h2-cts` for every trait
6. Aggregate all results into one TSV

## Input requirements

### BED files
Provide one BED file per cell type.

Minimum supported format:

```text
chr  start  end
```

A 4th column is allowed and will be ignored. For example:

```text
1  713739  714831  11.84087621
1  714876  715568  5.124349145
1  762078  763310  6.197571645
```

### Summary statistics
The workflow expects **LDSC-ready summary statistics** already munged, for example:

```text
sumstats/
├── trait1.sumstats.gz
├── trait2.sumstats.gz
└── ...
```

If `traits: []`, all matching files in `sumstats_dir` will be used automatically.

### Reference files
You need:
- chromosome-specific PLINK reference files (`.bed`, `.bim`, `.fam`)
- LDSC baseline annotations
- LDSC weights
- allele frequency files
- `ldsc.py`

## Configuration

Edit:

```text
config/config.yaml
```

and replace all placeholder paths.

Important fields:
- `celltypes`
- `bed_template`
- `plink_prefix_template`
- `ldsc_py`
- `baseline_prefix`
- `weights_prefix_template`
- `frq_prefix_template`
- `sumstats_dir`

## How path validation works

At startup, the workflow checks:
- BED files exist for all configured cell types
- `ldsc.py` exists
- the annotation script exists
- PLINK reference files exist for all chromosomes
- weights exist for all chromosomes
- frequency files exist for all chromosomes
- baseline files exist for all chromosomes
- summary statistics directory exists

This means configuration mistakes are caught early.

## Running the workflow

### Dry-run

```bash
snakemake --snakefile workflow/Snakefile -n -p
```

### Real run

```bash
snakemake --snakefile workflow/Snakefile --cores 4
```

You can increase `--cores` depending on your machine.

## Main outputs

```text
work/beds/{celltype}/{chr}.bed
ldscores/{celltype}/{chr}.annot.gz
ldscores/{celltype}/{chr}.l2.ldscore.gz
cts/all.ldcts
results/{trait}.cell_type_results.txt
results/all_cell_type_results.tsv
```

## Notes

- The workflow uses `ldsc.py --h2-cts`.
- The `.ldcts` file is built automatically from the LD score directories generated by the workflow.
- Munging is deliberately excluded so collaborators can use their own preferred GWAS preprocessing.
